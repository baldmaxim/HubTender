# Push

Закоммить все незакоммиченные изменения и запушь текущую ветку в origin.

## Шаги

1. Выполни параллельно:
   - `git status` — посмотреть незакоммиченные изменения и untracked-файлы (без флага `-uall`).
   - `git diff` — увидеть все staged и unstaged правки.
   - `git log --oneline -10` — свериться со стилем сообщений в репозитории.
   - `git branch --show-current` — определить текущую ветку.

2. Проанализируй изменения и составь коммит-сообщение **на русском**, в повелительном наклонении, кратко (1–2 предложения). Без приписок "Generated with Claude Code" и "Co-Authored-By".

3. **НЕ коммить** файлы, похожие на секреты (`.env`, `credentials.json`, `*.pem`, `*.key`). Если они попали в staging — предупреди пользователя и не коммить.

4. Добавь конкретные файлы по именам (не `git add -A`/`git add .`), сформируй коммит через HEREDOC:
   ```bash
   git commit -m "$(cat <<'EOF'
   Сообщение на русском.
   EOF
   )"
   ```

5. Проверь, отслеживает ли локальная ветка remote:
   - Если да — `git push`.
   - Если нет — `git push -u origin <branch>`.

6. **Запрет на force-push в main/master.** Если текущая ветка `main`/`master` — обычный push без `--force`.

7. После пуша выведи короткий итог: какой коммит создан (SHA + заголовок) и в какую ветку запушено.

## Если изменений нет

Если `git status` пуст — сообщи пользователю, что коммитить нечего, и не делай пустых коммитов. Если есть неотправленные коммиты — всё равно сделай `git push`.

## Если есть аргументы

Если пользователь передал аргументы в команду (`$ARGUMENTS`) — используй их как готовое сообщение коммита вместо автогенерации.
