## Table checksums (md5 of stable text aggregate)

Computed via `md5(string_agg(t::text, ',' ORDER BY pk))` on both OLD (at export time, written to manifest.json) and PROD (here at verify time).

Statuses:
- `match` — byte-identical between OLD and PROD ✓
- `mismatch` — data content differs ✗ → **VERIFY_FAILED**
- `jsonb_warning` — mismatch could be due to jsonb key reordering only — manual review, **VERIFY_OK_WITH_WARNINGS**
- `preexisting_rows` — PROD has more rows than OLD (pre-existing data); full-table checksum impossible, **VERIFY_OK_WITH_WARNINGS**
- `skipped` — missing OLD checksum (re-run :export)

> `auth.users` is intentionally **excluded** from this checksum section — its full-table text representation would include `encrypted_password`. See `AUTH_VERIFY_RESULT.md` for the password-safe row-by-row sha256 path.
