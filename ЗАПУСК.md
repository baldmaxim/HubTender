# Запуск портала на локальной машине

Пошаговая инструкция: от чистой системы до работающего портала в браузере.

---

## 1. Предустановленные программы

| Что | Зачем | Проверить |
|---|---|---|
| [Node.js 18+](https://nodejs.org/) | фронт (Vite + React) | `node -v` |
| [Docker Desktop](https://www.docker.com/products/docker-desktop/) | бэк (Go BFF) и Redis в контейнерах | `docker --version` |
| [git](https://git-scm.com/) | клонирование репозитория | `git --version` |

**Go ставить не нужно** — бэкенд собирается внутри Docker.

---

## 2. Склонировать репозиторий

```bash
git clone https://github.com/baldmaxim/HubTender.git
cd HubTender
```

---

## 3. Настроить переменные окружения

Скопировать шаблон и заполнить реальными значениями:

```bash
cp .env.example .env
```

Открыть `.env` и подставить:

| Переменная | Где взять |
|---|---|
| `VITE_SUPABASE_URL` | Supabase Dashboard → Project Settings → API → Project URL |
| `VITE_SUPABASE_PUBLISHABLE_KEY` | там же → `anon` / `publishable` key (публичный, можно класть в бандл) |
| `DATABASE_URL` | Supabase Dashboard → Database → Connection string → **Session pooler** (порт 5432) |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase Dashboard → API → `service_role` (секрет, **только серверный**) |
| `SUPABASE_JWT_SECRET` | Supabase Dashboard → Settings → JWT |

Остальные переменные (`VITE_API_URL`, `REDIS_URL`, фича-флаги) уже настроены с правильными дефолтами.

> ⚠️ `.env` в `.gitignore` — в репозиторий он не попадёт.
> ⚠️ Для бесплатного тарифа Supabase в `DATABASE_URL` обязательно **Session Pooler** (порт 5432), а не прямой хост `db.<ref>.supabase.co` — он отдаёт только IPv6, и Docker на Windows до него не достучится.

---

## 4. Установить зависимости фронта

```bash
npm install
```

---

## 5. Запустить бэкенд (Go BFF + Redis)

В **первом терминале** из корня репозитория:

```bash
docker compose up --build
```

Эта команда поднимает три сервиса:
- **api** — Go BFF на `http://localhost:3005`
- **redis** — кэш (внутри сети Docker)
- **caddy** — reverse proxy на `:80`/`:443` (нужен только для продакшна, локально можно игнорировать)

Проверить, что бэкенд поднялся:

```bash
curl http://localhost:3005/health      # должен вернуть OK
curl http://localhost:3005/health/db   # должен подтвердить коннект к Supabase
```

Если один из healthcheck'ов падает — смотреть логи в терминале с `docker compose`.

---

## 6. Запустить фронтенд (Vite dev-server)

Во **втором терминале** (первый оставить с бэкендом):

```bash
npm run dev
```

Vite автоматически откроет **http://localhost:5185** в браузере. Hot Module Replacement (HMR) включён — правки в `src/` применяются мгновенно.

---

## 7. Зайти в портал

На странице логина ввести креды пользователя из Supabase Auth. Если пользователей нет — создать через форму регистрации `/register` или через Supabase Dashboard → Authentication → Users.

После успешного входа — редирект на `/dashboard`.

---

## Остановка

- Фронт: `Ctrl+C` в терминале с `npm run dev`
- Бэк: `Ctrl+C` в терминале с `docker compose` (или `docker compose down` из другого терминала — остановит и удалит контейнеры)

## Troubleshooting

**Порт 5185 уже занят** — поменять в [vite.config.ts](vite.config.ts) на любой свободный (и обновить [.env](.env) → `CORS_ORIGINS`, и [docker-compose.yml](docker-compose.yml) → `CORS_ORIGINS`, и [playwright.config.ts](playwright.config.ts)).

**Порт 3005 уже занят** — поменять в [docker-compose.yml](docker-compose.yml) (`ports`, `PORT`, `healthcheck`), [Caddyfile](Caddyfile), [backend/internal/config/config.go](backend/internal/config/config.go) → `PORT` default, и `VITE_API_URL` в `.env`.

**`docker compose` падает на `api` с ошибкой про DATABASE_URL** — в `.env` должен быть URL через Session Pooler. Прямой хост `db.<ref>.supabase.co` не работает из Docker на Windows.

**Фронт поднимается, но данные не грузятся (401/403)** — проверить, что `VITE_SUPABASE_URL` и `VITE_SUPABASE_PUBLISHABLE_KEY` в `.env` принадлежат тому же проекту, что и `DATABASE_URL`. Не смешать pre-prod и старый prod проекты.

**В контейнере `api` логи `JWT verification failed`** — проверить `SUPABASE_JWT_SECRET` в `.env`. Если проект использует RS256 — вместо него подставить `SUPABASE_JWKS_URL` (пример — в [.env.example](.env.example)).
